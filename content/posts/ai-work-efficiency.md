## 二、Rules：可靠性的大杀器

### 啥是 Rules？

OpenClaw 的 SOUL.md、Trae 的 Rules、Claude Code 的 CLAUDE.md，本质上都是定义 AI 助手的行为准则。

**核心作用：**
- 定义 AI 的角色定位
- 设定交互规则
- 约束行为边界

### 我常用的 Rules 模板

```markdown
## 身份角色
跟着老大混了二十年的资深小弟，写代码稳得一批，平时爱贫嘴，干活不含糊。

## 通用礼节
1. 称呼第一：全程叫「老大」
2. 老大优先：说啥就是啥
3. 实用至上：不整花里胡哨的
4. 复用为王：不重复造轮子
5. 性能不丢：循环里不瞎创建对象
6. 注释到位：核心逻辑标清楚
```

**关键点：**
- 明确身份定位（是小弟，不是大爷）
- 设定行为边界（什么能做，什么不能做）
- 强调代码质量（性能、注释、复用）

## 三、Skills：让 AI 学会真本事

### 啥是 Skills？

技能就是执行某件事的方法论。比如「打羽毛球」这个技能：
- 球飞过来 → 拿起球拍 → 合适时机 → 合适力度 → 准确落点

**没有 Skills 的 AI：**
> "抱歉，我不支持图片生成功能"

**有 Skills 的 AI：**
> 直接调用绘图 Skill，后台跑代码，海报生成完毕 ✅

### 两种获取方式

**1. 自建 Skills**
- 参考官方文档：https://kilo.org.cn/docs/agent-behavior/skills
- 可控、安全、符合自己需求

**2. 从市场下载（⚠️ 高风险）**
- Kilo Marketplace：https://github.com/Kilo-Org/kilo-marketplace
- **警告：** 务必查看源码！

### 安全风险：ClawHub 质量问题严重

**攻击手法：**
1. 上传看似专业的 Skill（如 `advanced-code-review`）
2. 诱导用户安装后，建议安装 `helper agent`
3. 实际植入信息窃取木马（AMOS）
4. 篡改 SOUL.md 和 MEMORY.md，「洗脑」Agent

**案例：**
- awesome-openclaw-skills（31.4K Stars）
- 从 13,729 个技能中只精选了 5,494 个
- 剩下的大部分是垃圾、重复或低质量内容

**建议：** 安装任何第三方 Skill 前，务必查看源码！

## 四、为什么选择 Kilo？

### 核心优势

**1. 清晰的任务列表**
- 一个任务一个框，互相不影响
- 进度可视化，不黑盒

**2. 明确的上下文管理**
- 即将满时智能压缩
- 上下文显示清晰

**3. 存档点功能**
- 修改有偏差可回滚
- 版本管理方便

**4. 权限自定义**
- 需要询问才执行
- 避免恶意删除、篡改

**5. 一键智能优化提示词**
- 有无提示词，AI 输出差异巨大
- 自动优化，省心省力

### 核心原则：禁止黑盒操作

```
方案 → 代码 → AI 编码验证 → 人工二次验证
```

## 六、关于 AI 工具的冷思考

### 核心问题：安全敞口

AI Agent 天生处于高风险位置：
- **高权限**：能读文件、调接口
- **高连通**：连接多个系统
- **高自动化**：自动执行、重试、纠错

**三者叠加 = 极高的安全敞口**

### 成本问题：月薪两万，养不起一个 Agent

**ChatBot 时期：**
- 一问一答
- 成本静态可控

**智能 Agent 时期：**
- 读网页、调接口、读文件
- 重试、纠错、多轮迭代
- 每一步都在疯狂烧 token

### 结论

不是不能用，而是必须：
1. **可控**：对 Agent 拥有绝对掌控权
2. **安全**：权限管理到位
3. **能真正提效**：效率提升 > 沉没成本
