# 博客标签分类规范

## 分类（Categories）- 不超过 3 个

| 分类 | 用途 | 示例文章 |
|------|------|----------|
| `技术分享` | 技术对比、架构设计、选型指南 | 数据库选型、AI 工具使用心得 |
| `教程指南` | 步骤教学、部署指南、配置教程 | OpenClaw 部署、Hugo 博客搭建 |
| `工具推荐` | 工具介绍、效率提升 | 飞书导出工具 |

## 标签（Tags）- 每篇 2-3 个

| 标签 | 用途 | 说明 |
|------|------|------|
| `AI` | AI 相关工具、模型、应用 | OpenClaw、CodingPlan |
| `效率工具` | 提升工作效率的工具 | Kilo、Trae、feishu2md |
| `OpenClaw` | OpenClaw 专属标签 | 部署、配置、使用技巧 |
| `数据库` | 数据库相关 | MySQL、PG、TiDB 等 |
| `博客搭建` | Hugo、GitHub Pages 等 | 建站教程 |
| `飞书` | 飞书相关工具 | feishu2md |
| `教程指南` | 教学类文章 | 配合分类使用 |

## 使用原则

1. **分类只选 1 个** - 每篇文章只能有 1 个主分类
2. **标签 2-3 个** - 不要太多，覆盖核心主题即可
3. **复用已有标签** - 相同用处的文章用同一个标签，不要创建新标签
4. **先查后加** - 写新文章前先看看已有标签，优先复用

## 示例

```yaml
# AI 工具类文章
tags: ["AI", "效率工具"]
categories: ["技术分享"]

# 部署教程类文章
tags: ["OpenClaw", "教程指南"]
categories: ["教程指南"]

# 工具推荐类文章
tags: ["飞书", "效率工具"]
categories: ["工具推荐"]
```

---

**记住：** 标签分类是为了方便读者查找，不是越多越好！
